ImageIcon加载图片路径

本文总共1483个字 | 您是第位看到它们的小伙伴

一、问题

ImageIcon icon =new ImageIcon("pic.jpg");
执行结果是icon无法显示

二、解决

ImageIcon icon =new ImageIcon("pic.jpg");
很自然的认为当前类文件和图片在同一级目录下即可。
其实所谓的相对路径起点是工程的根目录，即project。
这行代码执行时在project目录下查找名为pic.jpg的文件，结果当然是找不到。
假如你的工程根目录为：project
你的图片文件放在：project/src/images/pic.jpg

所以正确的调用是：
'ImageIcon icon=new ImageIcon("src/images/pic.jpg");'
这行代码执行时在project目录下查找到了文件，当然也可以写成
'ImageIcon icon=new ImageIcon("./src/images/pic.jpg");'
注意两者的区别，要记住的是src是在project下的目录
总结起来就是一句话：所谓相对路径就是相对于工程根目录的位置

三、其他

第一种查看了一下ImageIcon的构造函数。
Pulbic ImageIcon(String filename)//参数可以是绝对路径也可以是相对路径
Public ImageIcon(URL url)
第二种通过URL来得到图片
'URL url =getClass().getResource("pic.jpg");'//当前编译后class文件所在目录查找
'ImageIcon icon =new ImageIcon(url);'
第三种可以通过图片的ID来添加图片
'ImageIcon icon =new ImageIcon("picture/pic_"+pictureID+".jpg");'

四、问题讨论

出现此类问题的原因大多是路径问题，往往在项目中运行正常，但是一旦打包成jar就不能正常
显示了。下面是一些常用解决方式：
### 一、通过使用内部资源文件的方式
    把所用的图片文件放在项目使用的包中，打包时把图片文件同时打进jar文件中：
    ImageIcon icon = new ImageIcon(类名.class.getResource("icons/logo.jpg");//icons与"类"同级,相对路径
    ImageIcon icon = new ImageIcon(任一类名.class.getResource("/icons/logo.jpg");//icons位于jar包的最外层，绝对路径
### 二、通过使用内部资源文件的方式
    把项目打包成jar后，再新建一文件夹，把项目中所有使用的图片文件和jar文件放在该目录下，
    只要代码对图片文件的应用路径正确，如：
    'ImageIcon icon = new ImageIcon("icons/logo.jpg");'
    这种方式下，图片文件和类文件不是一块放在包中，类文件在jar中，而图片文件夹在项目根目录下。
